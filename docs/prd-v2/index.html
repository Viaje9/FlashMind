FlashMind PRD v2

版本日期作者變更說明
2.02026-01-11—依據 prototype 重構為傳統 PRD 格式

1. 產品願景

讓每個人都能用最少的時間，建立有效的長期記憶。

2. 問題陳述

痛點現有解決方案的問題
建卡成本高Anki 需手動輸入所有內容，耗時且容易放棄
學習曲線陡Anki 介面複雜、設定繁多，新手難以上手
行動體驗差許多工具以桌面為主，手機體驗不佳
演算法過時SM-2 演算法已有 30 年歷史，排程效率不如新方法

FlashMind 透過 AI 自動生成內容、簡潔的行動優先介面、以及 FSRS 演算法，解決上述痛點。

3. 目標用戶 (Personas)

Persona 1：語言學習者 — 小美

屬性描述
背景25 歲，上班族，每天通勤 40 分鐘
目標準備多益考試，需要背誦大量單字
痛點沒時間手動製作卡片；在擁擠的捷運上無法使用筆電
使用情境通勤時用手機學習 15-20 分鐘
期望輸入單字後自動產生例句，手機操作流暢

Persona 2：專業考試準備者 — 阿凱

屬性描述
背景30 歲，準備專業證照考試
目標記憶大量專業術語與定義
痛點資料量大，需要有效率的複習排程
使用情境週末集中學習 1-2 小時
期望系統自動安排複習順序，專注在快忘記的卡片

4. 競品分析與差異化

特性AnkiQuizletFlashMind
AI 生成內容❌ 需手動⚠️ 有限支援✅ 一鍵生成詞義/例句
學習曲線❌ 陡峭✅ 簡單✅ 簡單
行動體驗⚠️ 可用但非最佳化✅ 良好✅ 行動優先設計
間隔重複演算法SM-2無/簡易FSRS (更高效率)
離線支援✅⚠️ 付費✅ (PWA)
價格免費免費增值TBD

核心差異化定位：FlashMind = Anki 的效率 + Quizlet 的易用性 + AI 的便利性

5. 產品形態

  • 類型：Progressive Web App (PWA)
  • 支援平台：iOS Safari、Android Chrome、桌面瀏覽器
  • 最小螢幕寬度：360px
  • 離線支援：透過 Service Worker 快取核心功能

6. 用戶故事與驗收標準

6.1 帳號與登入

US-001：新用戶註冊

As a 新用戶 I want to 快速註冊帳號 So that 我可以開始使用 FlashMind

驗收標準：

  • 支援 Email 註冊
  • 支援第三方登入（Google）
  • 註冊成功後自動登入並導向主介面
  • 顯示服務條款與隱私權政策連結

US-002：現有用戶登入

As a 現有用戶 I want to 登入我的帳號 So that 存取我的牌組與學習紀錄

驗收標準：

  • 支援 Email + 密碼登入
  • 支援第三方登入（Google）
  • 登入成功後導向主介面
  • 登入失敗顯示錯誤訊息

US-003：登出帳號

As a 已登入用戶 I want to 登出帳號 So that 保護我的資料安全

驗收標準：

  • 在設定頁提供登出選項
  • 登出後清除本地登入狀態
  • 登出後導向歡迎頁

6.2 牌組管理

US-010：檢視牌組列表

As a 用戶 I want to 檢視我所有的牌組 So that 選擇要學習或管理的牌組

驗收標準：

  • 顯示所有牌組的列表
  • 每個牌組顯示：名稱、新卡數、待複習數、完成數
  • 顯示學習進度條與百分比
  • 提供「開始學習」快捷入口
  • 空狀態顯示引導建立牌組

US-011：搜尋牌組

As a 擁有多個牌組的用戶 I want to 搜尋牌組 So that 快速找到目標牌組

驗收標準：

  • 提供搜尋輸入框
  • 即時過濾牌組列表（輸入時即過濾）
  • 無結果時顯示「找不到符合的牌組」

US-012：建立新牌組

As a 用戶 I want to 建立新牌組 So that 組織我的學習內容

驗收標準：

  • 點擊右下角 + 按鈕開啟建立表單
  • 必填：牌組名稱
  • 可設定每日新卡數（預設 20，步進 5，範圍 5-100）
  • 可設定每日複習數（預設 100，步進 10，範圍 10-500）
  • 顯示提示：「可於牌組設定調整」
  • 儲存成功後導向牌組卡片列表
  • 取消時返回主介面，不儲存任何資料

US-013：編輯牌組設定

As a 用戶 I want to 修改牌組的學習設定 So that 調整學習計畫

驗收標準：

  • 可修改牌組名稱
  • 可修改每日新卡數與複習數
  • 儲存後立即生效於下次學習

US-014：刪除牌組

As a 用戶 I want to 刪除不需要的牌組 So that 保持牌組列表整潔

驗收標準：

  • 刪除前顯示確認對話框
  • 確認對話框說明將刪除所有卡片與學習紀錄
  • 刪除後返回主介面

6.3 卡片管理

US-020：檢視牌組內卡片

As a 用戶 I want to 檢視牌組內所有卡片 So that 管理學習內容

驗收標準：

  • 顯示牌組摘要：新卡、待複習、建立時間、上次複習
  • 顯示卡片列表，每張卡片顯示正面文字與解釋摘要
  • 提供「開始學習」按鈕
  • 空狀態顯示引導新增卡片

US-021：搜尋卡片

As a 用戶 I want to 搜尋牌組內的卡片 So that 快速找到特定卡片

驗收標準：

  • 提供搜尋輸入框
  • 搜尋範圍包含正面文字與背面解釋
  • 即時過濾卡片列表

US-022：新增卡片

As a 用戶 I want to 新增快閃卡 So that 增加學習內容

驗收標準：

  • 點擊右下角 + 按鈕開啟新增表單
  • 正面：輸入單字/片語/問題
  • 背面：支援多筆詞義區塊
  • 每筆詞義包含：中文解釋、英文例句、中文例句翻譯
  • 可新增/刪除詞義區塊
  • 英文例句提供播放語音按鈕
  • 頁首提供「取消」「儲存」
  • 儲存成功後返回卡片列表

US-023：AI 生成卡片內容

As a 用戶 I want to 一鍵生成詞義與例句 So that 快速建立完整卡片

驗收標準：

  • 提供「AI 生成」按鈕
  • 輸入正面文字後可觸發生成
  • 生成過程顯示載入狀態
  • 生成結果自動填入背面欄位
  • 用戶可編輯/刪除生成內容
  • 生成失敗顯示錯誤訊息與重試選項

US-024：編輯卡片

As a 用戶 I want to 編輯現有卡片 So that 修正或補充內容

驗收標準：

  • 點擊卡片列表項目的編輯按鈕進入編輯
  • 可修改正面與所有背面欄位
  • 儲存成功後返回卡片列表

US-025：刪除卡片

As a 用戶 I want to 刪除不需要的卡片 So that 移除錯誤或重複內容

驗收標準：

  • 點擊卡片列表項目的刪除按鈕
  • 刪除前顯示確認對話框
  • 刪除後更新牌組統計

6.4 學習模式

US-030：開始學習

As a 用戶 I want to 開始學習牌組 So that 記憶卡片內容

驗收標準：

  • 顯示牌組名稱
  • 顯示學習進度（例：12/50）與進度條
  • 依 FSRS 演算法排序卡片（新卡 + 待複習）
  • 優先顯示待複習卡片，再顯示新卡
  • 每日學習數量不超過設定上限

US-031：翻卡學習

As a 用戶 I want to 查看卡片正面後翻到背面 So that 測試自己是否記得

驗收標準：

  • 初始顯示卡片正面（單字/問題）
  • 點擊/滑動可翻到背面
  • 背面顯示：中文解釋、例句、例句翻譯
  • 翻卡動畫流暢（60fps）

US-032：回答評分

As a 用戶 I want to 透過滑動手勢評分卡片 So that 快速記錄學習狀態並更新複習排程

驗收標準：

  • 翻到背面後支援三種滑動手勢：
    • 右滑 = 知道
    • 上滑 = 不熟
    • 左滑 = 不知道
  • 滑動後記錄結果並進入下一張卡片
  • 根據回答更新 FSRS 排程（知道的間隔最長，不知道的間隔最短）
  • 「不知道」的卡片會在本次學習中再次出現
  • 滑動時顯示視覺回饋（卡片移動、背景色變化）
  • 支援返回按鈕，讓用戶修改評分

US-033：卡片操作

As a 用戶 I want to 在學習中收藏卡片或播放發音 So that 標記重要內容並加強聽力

驗收標準：

  • 提供收藏按鈕，點擊後標記為收藏
  • 提供播放發音按鈕，點擊後播放正面文字的語音
  • 收藏狀態在卡片列表中可見

US-034：完成學習

As a 用戶 I want to 完成今日學習 So that 知道學習成果

驗收標準：

  • 所有卡片完成後顯示結束畫面
  • 顯示本次學習統計：學習張數、各評分分布（知道/不熟/不知道）
  • 提供「返回牌組」按鈕

6.5 設定

US-040：學習提醒

As a 用戶 I want to 設定每日學習提醒 So that 不會忘記學習

驗收標準：

  • 可開啟/關閉提醒
  • 可設定提醒時間
  • 透過瀏覽器通知推送

US-041：管理帳戶

As a 用戶 I want to 管理我的帳戶資訊 So that 更新個人資料

驗收標準：

  • 可檢視帳戶 Email
  • 可修改密碼（Email 用戶）
  • 可刪除帳戶（顯示確認警告）

11. 假設與限制

假設

  • 用戶主要使用繁體中文
  • 用戶有穩定的網路連線（離線為次要場景）
  • AI 生成服務有足夠配額支援用量

限制

  • AI 生成詞義/例句需有可用模型與 API 配額
  • 語音播放依賴外部 TTS 服務
  • 首版僅支援繁體中文介面
  • PWA 推播通知需用戶授權

技術依賴

功能依賴服務
AI 生成OpenAI API / Claude API
語音播放Web Speech API / 外部 TTS
第三方登入Google OAuth
推播通知Web Push API

12. 風險評估

風險可能性影響緩解措施
AI 生成品質不佳中高用戶可編輯生成內容；持續優化 prompt
AI API 成本過高中中設定每日生成上限；考慮本地模型
FSRS 實作複雜度低高使用現有開源實作 (ts-fsrs)
離線同步衝突中中採用 last-write-wins 策略；顯示衝突提示
瀏覽器相容性低中限定支援 Safari 15+、Chrome 90+

13. 里程碑

M1：核心功能

  • 帳號登入/註冊（含第三方登入）
  • 牌組 CRUD
  • 卡片 CRUD
  • 基礎 UI 完成

M2：學習功能

  • 學習模式（翻卡、評分）
  • FSRS 演算法整合
  • 學習進度追蹤
  • 語音播放

M3：智慧功能

  • AI 生成詞義/例句
  • 學習提醒通知
  • 設定頁完善
  • PWA 離線支援

M4：優化與擴展

  • 學習數據匯出
  • 效能優化
  • 無障礙改善
  • 使用者回饋迭代
