Post-PRD Checklist

完成 PRD-v2 後，下一步需要準備的文件與交付物清單。

第一階段：設計與架構（必須）

☐ UI/UX 設計文件 (Figma / 設計稿)

用途：將 PRD 中的用戶故事具體化為視覺設計與交互流程

重點：

  • 學習模式的三種滑動手勢（右滑知道、上滑不熟、左滑不知道）
  • AI 生成內容的 loading 狀態與結果展示
  • 歡迎/登入頁面設計
  • 行動優先佈局（360px 起）
  • 深色模式支援

交付物：

  • 所有頁面的高保真設計稿
  • 交互流程圖（從歡迎頁到學習完成）
  • 元件庫（按鈕、表單、卡片等）
  • 無障礙檢查清單（對比度、可點擊區域大小）

負責人：設計師

☐ 資料模型文件 (ER 圖、Schema 定義)

用途：定義資料庫結構，後端開發的基礎

重點：

  • User（帳號、密碼、第三方登入）
  • Deck（牌組、每日新卡數/複習數設定）
  • Card（正面、背面、詞義、例句等）
  • LearningRecord（評分、時間戳、FSRS 狀態）
  • FSRS 相關欄位（difficulty、stability、next_review 等）

交付物：

  • ER 圖（實體關係圖）
  • 表格 Schema（欄位名、型別、約束）
  • 索引策略（用於查詢最佳化）
  • 資料遷移與備份方案

負責人：後端工程師 / 產品經理

☐ API 設計文件 (OpenAPI / 接口規格)

用途：定義前後端通信的接口

重點：

  • 認證接口（註冊、登入、第三方登入）
  • 牌組相關（建立、編輯、刪除、列表）
  • 卡片相關（建立、編輯、刪除、搜尋）
  • 學習接口（開始學習、提交回答、完成學習）
  • AI 生成接口（呼叫 AI 生成詞義/例句）
  • 設定接口（提醒時間、帳戶管理）

交付物：

  • OpenAPI 3.0 規格 (YAML/JSON)
  • 每個 endpoint 的 request/response 範例
  • 錯誤碼定義
  • 認證與授權方案（JWT/OAuth）
  • 速率限制政策

負責人：後端工程師

第二階段：開發與測試（重要）

☐ 技術規格書 (Technical Specification)

用途：詳細的開發實現方案、技術棧選擇、架構設計

重點：

  • 技術棧選擇（前端框架、後端框架、資料庫、AI API）
  • 系統架構圖（前端、後端、資料庫、外部服務）
  • FSRS 演算法實作方案
  • AI 生成的 prompt 設計與成本評估
  • 認證流程（OAuth 與 JWT）
  • 快取策略（Redis、LocalStorage）
  • 檔案結構與命名規範

交付物：

  • 系統架構圖
  • 技術決策與權衡分析
  • 程式碼結構與模組化方案
  • 開發環境設置指南
  • 相依套件清單

負責人：技術主管 / 後端工程師

☐ 測試計畫書 (QA Test Plan)

用途：測試策略、測試場景、驗收標準

重點：

  • 功能測試場景（每個 User Story 對應的測試案例）
  • 三種滑動手勢的邊界情況
  • AI 生成的品質驗收標準
  • 效能測試（LCP < 2.5s、INP < 200ms）
  • 相容性測試（iOS Safari、Android Chrome）
  • 離線模式測試

交付物：

  • 完整的測試案例清單（含預期結果）
  • 自動化測試計畫（單元測試、整合測試、E2E 測試）
  • 效能基準測試
  • UAT（用戶驗收測試）清單

負責人：QA / 測試工程師

第三階段：專案管理與運維（優化）

☐ 開發時程表詳版 (Detailed Roadmap)

用途：里程碑 M1-M4 的詳細任務拆分與時程

交付物：

  • 任務清單（JIRA / Linear 票證）
  • 依賴關係圖（哪些任務需要先完成）
  • 里程碑時間表
  • 風險與 blockers 識別

負責人：項目經理

☐ 部署與運維計畫 (DevOps / Operations Plan)

用途：環境設置、CI/CD、監控、備份策略

交付物：

  • 開發、測試、上線環境設置
  • CI/CD 流程（自動化測試、部署）
  • 監控與告警規則（錯誤率、回應時間）
  • 備份與災難恢復計畫
  • 安全性檢查清單（HTTPS、密碼雜湊、SQL injection 防護）

負責人：DevOps / 後端工程師

FlashMind 特殊考量

優先處理項目

由於 FlashMind 有以下特殊需求，建議優先完成：

  1. 三種滑動手勢互動設計

    • 需要 UI 設計詳細說明與動畫規格
    • 需要 API 支援三種評分狀態

  2. FSRS 演算法實作

    • 建議使用現有開源套件（如 ts-fsrs）
    • 需要在技術規格書中明確定義參數

  3. AI 生成功能成本評估

    • 需要在技術規格書中分析成本
    • 建議在測試計畫中定義品質驗收標準

檢查清單模板

📋 如何使用本清單

  • 完成資料模型
  • 完成 API 設計
  • 完成 UI/UX 設計
  • 完成技術規格書
  • 完成測試計畫
  • 建立開發任務清單
  • 設置 CI/CD 流程
  • 進入 M1 開發階段
