## Why

目前牌組卡片列表已支援透過過濾碼篩選 Deck 內卡片，但當牌組同時包含正面與反面卡時，使用者無法快速只看其中一種卡面。這讓檢查反向學習內容、排查卡片狀態或專注編輯單一方向卡片時效率不佳。

## What Changes

- 在牌組卡片列表的既有過濾區新增卡面方向下拉選單，支援選擇顯示正面卡片或反面卡片
- 當牌組只存在正面卡片、沒有任何反面卡片時，不顯示卡面方向下拉選單，避免無效操作
- 將卡片列表查詢與前端篩選狀態擴充為可攜帶卡面方向條件，讓列表、空狀態與重新整理結果一致
- 補充牌組卡片管理規格，明確定義方向篩選的可見條件、可選值與行為

### New Capabilities

### Modified Capabilities

- `card-management`: 調整牌組卡片列表篩選能力，新增正面/反面方向篩選與條件式顯示規則

## Impact

- **前端**：`apps/web` 的牌組卡片列表頁、篩選表單狀態與列表資料請求需新增方向選項與條件式渲染
- **後端/API**：若現有卡片列表 API 尚未支援方向條件，需在 `apps/api` 與 OpenAPI 契約補上對應查詢參數與過濾邏輯
- **規格/測試**：需更新 `card-management` spec，並補上前端 domain/store 與後端 service 的方向篩選測試案例
