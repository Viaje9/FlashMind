## ADDED Requirements

### Requirement: 收藏包聊天可建議新增缺少的單字卡

系統 SHALL 在新增收藏聊天中，針對句子或語塊裡重要但不存在於使用者單字卡的英文單字，回傳可新增的單字卡候選。

#### Scenario: 句子包含未建立的主要單字

- **WHEN** 使用者輸入一句中英文內容並要求 AI 整理可收藏表達
- **AND** Codex agent 判斷句子中有重要英文單字不存在於使用者既有卡片
- **THEN** 回應 SHALL 包含單字卡候選
- **AND** 候選 SHALL 包含正面文字、中文解釋、英文例句與中文例句翻譯

#### Scenario: 單字已存在於卡片

- **WHEN** 使用者輸入內容中的主要單字已能對應到使用者既有卡片
- **THEN** 系統不得將該單字作為可新增單字候選
- **AND** 系統 SHALL 優先將既有卡片作為句子或語塊候選的來源卡片

#### Scenario: 單字不是主要學習重點

- **WHEN** 句子中存在功能字、冠詞、介系詞或對理解表達幫助有限的單字
- **THEN** Codex agent 不得為了湊數產生單字卡候選
- **AND** 回應 SHALL 只保留對句子或語塊理解有實質幫助的建議單字

### Requirement: 單字候選須由使用者確認後才建立

系統 SHALL 將 AI 回傳的單字卡視為候選，只有在使用者點擊加號並完成新增卡片流程後才正式建立卡片。

#### Scenario: AI 回應包含單字候選

- **WHEN** Codex agent 回傳單字卡候選
- **THEN** 系統 SHALL 保存該候選於聊天回應 metadata 或回傳資料
- **AND** 系統不得在此階段建立 Card 或 CardMeaning 資料

#### Scenario: 使用者點擊單字候選加號

- **WHEN** 使用者點擊尚未新增的單字候選加號
- **THEN** 系統 SHALL 先要求使用者選擇要加入的牌組
- **AND** 系統不得在使用者選擇牌組前建立卡片

#### Scenario: 使用者取消新增流程

- **WHEN** 使用者在選牌組或新增快閃卡表單中取消
- **THEN** 系統 SHALL 保留原聊天內容
- **AND** 單字候選 SHALL 維持可新增狀態
- **AND** 系統不得建立卡片

### Requirement: 成功新增單字卡後更新候選狀態

系統 SHALL 在卡片建立成功後更新新增收藏聊天中的單字候選狀態，讓使用者知道該單字已加入指定牌組。

#### Scenario: 建立卡片成功

- **WHEN** 使用者完成新增快閃卡表單並 API 回傳成功
- **THEN** 單字候選卡 SHALL 顯示已加入狀態
- **AND** 單字候選卡 SHALL 顯示或保留所加入的牌組資訊
- **AND** 該單字候選不得再顯示加號

#### Scenario: 建立卡片失敗

- **WHEN** 新增快閃卡 API 回傳驗證錯誤或伺服器錯誤
- **THEN** 系統 SHALL 顯示可理解的錯誤訊息
- **AND** 單字候選 SHALL 維持可重新嘗試的狀態
