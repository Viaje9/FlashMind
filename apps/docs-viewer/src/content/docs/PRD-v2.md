## 1. 產品願景

讓每個人都能用最少的時間，建立有效的長期記憶。

---

## 2. 問題陳述

| 痛點 | 現有解決方案的問題 |
|------|-------------------|
| 建卡成本高 | Anki 需手動輸入所有內容，耗時且容易放棄 |
| 學習曲線陡 | Anki 介面複雜、設定繁多，新手難以上手 |
| 行動體驗差 | 許多工具以桌面為主，手機體驗不佳 |
| 演算法過時 | SM-2 演算法已有 30 年歷史，排程效率不如新方法 |

FlashMind 透過 **AI 自動生成內容**、**簡潔的行動優先介面**、以及 **FSRS 演算法**，解決上述痛點。

## 4. 競品分析與差異化

| 特性 | Anki | Quizlet | FlashMind |
|------|------|---------|-----------|
| AI 生成內容 | ❌ 需手動 | ⚠️ 有限支援 | ✅ 一鍵生成詞義/例句 |
| 學習曲線 | ❌ 陡峭 | ✅ 簡單 | ✅ 簡單 |
| 行動體驗 | ⚠️ 可用但非最佳化 | ✅ 良好 | ✅ 行動優先設計 |
| 間隔重複演算法 | SM-2 | 無/簡易 | FSRS (更高效率) |
| 離線支援 | ✅ | ⚠️ 付費 | ✅ (PWA) |
| 價格 | 免費 | 免費增值 | TBD |

**核心差異化定位**：FlashMind = Anki 的效率 + Quizlet 的易用性 + AI 的便利性

## 5. 產品形態

- **類型**：Progressive Web App (PWA)
- **支援平台**：iOS Safari、Android Chrome、桌面瀏覽器
- **最小螢幕寬度**：360px
- **離線支援**：透過 Service Worker 快取核心功能

- [ ] 支援第三方登入（Google）

#### US-011：搜尋牌組
> **As a** 擁有多個牌組的用戶
> **I want to** 搜尋牌組
> **So that** 快速找到目標牌組

**驗收標準：**
- [ ] 提供搜尋輸入框
- [ ] 即時過濾牌組列表（輸入時即過濾）

#### US-012：建立新牌組
> **As a** 用戶
> **I want to** 建立新牌組
> **So that** 組織我的學習內容

#### US-013：編輯牌組設定

**驗收標準：**
- [ ] 可修改牌組名稱
- [ ] 可修改每日新卡數與複習數
- [ ] 儲存後立即生效於下次學習

#### US-014：刪除牌組
> **As a** 用戶
> **I want to** 刪除不需要的牌組
> **So that** 保持牌組列表整潔

**驗收標準：**
- [ ] 刪除前顯示確認對話框
- [ ] 確認對話框說明將刪除所有卡片與學習紀錄
- [ ] 刪除後返回主介面

### 6.4 學習模式

#### US-030：開始學習
> **As a** 用戶
> **I want to** 開始學習牌組
> **So that** 記憶卡片內容

**驗收標準：**
- [ ] 顯示牌組名稱
- [ ] 顯示學習進度（例：12/50）與進度條
- [ ] 依 FSRS 演算法排序卡片（新卡 + 待複習）
- [ ] 優先顯示待複習卡片，再顯示新卡
- [ ] 每日學習數量不超過設定上限

#### US-031：翻卡學習
> **As a** 用戶
> **I want to** 查看卡片正面後翻到背面
> **So that** 測試自己是否記得

**驗收標準：**
- [ ] 初始顯示卡片正面（單字/問題）
- [ ] 點擊/滑動可翻到背面
- [ ] 背面顯示：中文解釋、例句、例句翻譯
- [ ] 翻卡動畫流暢（60fps）

#### US-032：回答評分
> **As a** 用戶
> **I want to** 透過滑動手勢評分卡片
> **So that** 快速記錄學習狀態並更新複習排程

**驗收標準：**
- [ ] 翻到背面後支援三種滑動手勢：
  - 右滑 = 知道
  - 上滑 = 不熟
  - 左滑 = 不知道
- [ ] 滑動後記錄結果並進入下一張卡片
- [ ] 根據回答更新 FSRS 排程（知道的間隔最長，不知道的間隔最短）
- [ ] 「不知道」的卡片會在本次學習中再次出現
- [ ] 滑動時顯示視覺回饋（卡片移動、背景色變化）
- [ ] 支援返回按鈕，讓用戶修改評分

#### US-033：卡片操作
> **As a** 用戶
> **I want to** 在學習中收藏卡片或播放發音
> **So that** 標記重要內容並加強聽力

**驗收標準：**
- [ ] 提供收藏按鈕，點擊後標記為收藏
- [ ] 提供播放發音按鈕，點擊後播放正面文字的語音
- [ ] 收藏狀態在卡片列表中可見

#### US-034：完成學習
> **As a** 用戶
> **I want to** 完成今日學習
> **So that** 知道學習成果

**驗收標準：**
- [ ] 所有卡片完成後顯示結束畫面
- [ ] 顯示本次學習統計：學習張數、各評分分布（知道/不熟/不知道）
- [ ] 提供「返回牌組」按鈕

## 11. 假設與限制

### 假設
- 用戶主要使用繁體中文
- 用戶有穩定的網路連線（離線為次要場景）
- AI 生成服務有足夠配額支援用量

### 限制
- AI 生成詞義/例句需有可用模型與 API 配額
- 語音播放依賴外部 TTS 服務
- 首版僅支援繁體中文介面
- PWA 推播通知需用戶授權

### 技術依賴
| 功能 | 依賴服務 |
|------|----------|
| AI 生成 | OpenAI API / Claude API |
| 語音播放 | Web Speech API / 外部 TTS |
| 第三方登入 | Google OAuth |
| 推播通知 | Web Push API |

## 12. 風險評估

| 風險 | 可能性 | 影響 | 緩解措施 |
|------|--------|------|----------|
| AI 生成品質不佳 | 中 | 高 | 用戶可編輯生成內容；持續優化 prompt |
| AI API 成本過高 | 中 | 中 | 設定每日生成上限；考慮本地模型 |
| FSRS 實作複雜度 | 低 | 高 | 使用現有開源實作 (ts-fsrs) |
| 離線同步衝突 | 中 | 中 | 採用 last-write-wins 策略；顯示衝突提示 |
| 瀏覽器相容性 | 低 | 中 | 限定支援 Safari 15+、Chrome 90+ |
